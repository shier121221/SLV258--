结合你提供的 ST7789V 驱动 IC + 3.2 寸 I8080 接口 TFT 屏 两份资料，以及此前为数字毫秒计设计的 UI 需求，我先提炼硬件核心参数、再基于 320(W)×240(H) 标准分辨率，逐像素落地主界面完整设计（完全适配该屏幕驱动、接口、触控特性），同时补充硬件驱动、色彩、刷新、交互的适配要点，方便你直接用于绘图、UI 切分、单片机程序开发。
一、屏幕 & 驱动 核心硬件参数提炼（开发前置依据）
1. 基础规格
屏幕尺寸：3.2 英寸 TFT 彩色液晶屏
驱动芯片：ST7789V
物理分辨率：横向 320px × 纵向 240px（横屏）（固定画布尺寸，所有 UI 按此像素设计）
接口类型：标准 I8080 8 位并行接口（工业模块主流配置，MCU 直连 3.3V IO，无需电平转换）
触控类型：配套电阻触摸屏（仅支持点击操作，不支持滑动 / 拖拽，UI 全部采用点击交互）
2. 色彩与显示能力
支持色彩模式：
RGB565（65K 色，推荐使用，接口速度快、MCU 资源占用低，仪器界面首选）
RGB666（262K 色）
内置显存、局部刷新、垂直滚动、硬件亮度 PWM 调节，适合仪器静态 / 简易动态界面。
3. 电气 & 使用场景
IO 电压：1.65~3.3V，适配常规 3.3V 单片机系统
工作温度范围宽，适配实验室 / 教学设备场景
接口引脚：CS片选、DC指令/数据、WR写、RD读、RST复位、DB0~DB7数据口，标准 8080 时序。
4. UI 设计约束（硬性规则）
交互：仅点击，禁止滑动、拖拽（电阻屏特性）；
色彩：优先低饱和度工业色，高对比度，保证教室强光下可视；
字体：选用屏驱动通用点阵黑体，字号不宜过小；
刷新：优先局部刷新（利用 ST7789 显存，减少全屏刷新、防闪屏）。

五、ST7789+I8080 驱动 & 界面适配关键要点
结合屏幕手册，给出程序开发必配参数，避免屏驱动异常、显示错位：
1. 接口配置（8 位 I8080）
接口模式：IM [3:0] 配置为 0000（标准 8 位 8080 接口）
数据位宽：8 位并行 DB0~DB7
控制引脚：CS、DC (RS)、WR、RD、RST 常规 8080 时序
2. 色彩模式（推荐）
指令 0x3A 设置为 0x05 → 启用 RGB565 (65K 色)
优势：速度快、MCU 内存占用小，仪器界面首选。
3. 显示方向（横屏 320×240）
通过 0x36(MADCTL) 指令设置横屏扫描方向，保证 UI 坐标和设计完全对应，避免画面旋转 / 镜像。
4. 刷新优化（ST7789 显存优势）
顶部状态栏、功能卡片、按键分区局部刷新，禁止全屏刷屏（防闪烁、提升响应速度）；
仅在页面切换、状态变更时刷新对应区域，静态内容常驻显存。
5. 电阻触控适配
所有操作仅做点击识别，屏蔽滑动、长按（如需长按可单独做 1s 延时）；
触控采样 + 软件防抖（20~30ms），消除工业环境下的触点漂移；
触摸坐标和屏幕像素做精准映射（320×240 对应触控 XY 轴）。
6. LED 指示灯联动（原设备硬件保留）
屏幕显示「待机」：板载绿灯常亮
进入计时功能：绿灯闪烁
故障 / 超限告警：红灯亮起
时标输出功能：黄灯亮起
